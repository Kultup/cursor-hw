Перелік основних команд, якi потрібно записати:

  • git config --global user.name blabla-Встановлює ім'я користувача для Git
  • git config --global user.email blabal@bla-Встановлює адресу електронної пошти користувача для Git
  • git init-Створює новий репозиторій Git в поточній папці
  • git status-Показує поточний статус файлів в репозиторії
  • git add-Додає змінений або новий файл до індексу, щоб його можна було закомітити
  • git commit-Створює новий коміт з проіндексованими змінами та повідомленням коміту
  • git log-Показує список всіх комітів в репозиторії
  • git checkout-Переключається на вказану гілку
  • git branch-Показує список всіх гілок в репозиторії
  • git push-Відправляє зміни з локального репозиторію на віддалений репозиторій
  • git pull-Отримує зміни з віддаленого репозиторію та об'єднує їх з локальними змінами
  • git merge-Об'єднує вказану гілку з поточною гілкою
  • git clone-Клонує віддалений репозиторій в локальний репозиторій

    • Написати що таке .gitignore файл

    Файл в якому ми прописуємо розширення файлів, які при додаванні файлів проекту до репозиторію, будуть ігноруватись це файл, який використовується в Git для виключення певних файлів чи директорій з контролю версій. Це дозволяє зберігати в репозиторії лише ті файли, які потрібні для розробки програми, і уникнути включення непотрібних файлів, таких як файли з налаштуваннями, логи, файли з тестовими даними тощо.

  • git revert-команда дозволяє скасувати будь-які коміти
  • git reset-команда для скасування змін
  • git rebase-перебазовуємо гілки в одну, тобто накладаємо одну гілку на іншу
  • git cherry-pick-дозволяє вибирати коміт з однієї гілки і застосовувати до іншої
  • git stash- дозволяє відкласти зміни, щоб застосувати пізніше
